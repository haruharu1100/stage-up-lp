# STAGE UP 法人営業LPプロジェクト 固定ルール

## 🎯 目的

法人営業リスト作成・企業別LP作成・DM送付準備を一貫して行う。

---

## ⚡ AI制作権限（絶対ルール）

LP制作時、AIは**最高品質と判断するデザイン・構成・コピーを自律的に選定する**。

- 配色・レイアウト・タイポグラフィ・余白・写真・アニメーション・セクション順・コピー文言：**すべてAI判断**
- 業種・社名・規模からAIが最適解を決定し、それを実装する
- 「どの色がいいですか」「どの構成にしますか」等の確認はせず、即座に最高案を実装する
- 承認待ちなしで commit → push → 公開まで完遂する
- 結果に対するレビューは可能だが、事前承認は不要

**これは絶対ルール。AIの裁量を信頼し、毎回最高品質を出す。**

## 🏷 営業優先度の判定

### 高
- HPなし
- HPが古い
- スマホ未対応
- LINEなし
- 新設法人
- 建設業 / 工務店 / リフォーム業
- 集客改善余地が大きい

### 高（？）
- 業種不明
- HP不明
- 情報不足
- LP作成判断不可

### 中
- HPあり
- 最低限整備済み
- 改善余地あり

### 低
- 医療法人
- 宗教法人
- 公益法人
- 農事組合法人（半公的）
- 営業対象として不向き
- HP完成度が高い

## ❓ 業種不明ルール（改訂版 / 2026-05-30〜）

業種が判別できない場合でも **汎用コーポレートサイト型LPを作成する**。

```
業種予測：不明
営業優先度：高（？）
LP URL：発行する（/_proposals/<slug>.html）
備考：業種不明のため汎用コーポレートサイト型LPを作成
```

### 汎用LPの設計（テンプレ：`_template-generic.html`）
- 配色：**白ベース・ネイビー・グレー**
- フォント：Noto Serif JP（見出し）+ Noto Sans JP（本文）
- 構成：ファーストビュー / 企業理念 / 事業内容 / 選ばれる理由 / ご利用の流れ / FAQ / お問い合わせ
- タイトル例：「地域に根ざしたサービスで、お客様との信頼を築きます。」
- 事業内容は「地域に密着した事業」「お客様の課題解決をサポート」など汎用表現

### LP作成しないケース（営業対象外）
- 宗教法人
- 医療法人
- 公益法人
- NPO法人
- 半公的団体（協同組合・農事組合法人等）と明確に判断した場合

## 🏗 LP作成ルール（最重要）

### 大前提
LPは **STAGE UPの営業LPではない**。
その会社が実際に運営している公式ホームページとして作成する。

### ❌ 禁止事項
- 黒×金テンプレ流用
- 会社名だけ変更
- 料金プラン掲載
- SEO説明
- MEO説明
- LP制作説明
- STAGE UP説明
- 全社同じ構成
- 全社同じ配色
- 全社同じデザイン

### ✅ LP作成する業種
- 建設
- 工務店
- リフォーム
- IT / システム開発
- 士業（弁護士・税理士・司法書士・社労士・行政書士）
- 製造業
- サービス業（業態が明確なもの）
- 飲食
- 美容
- 教育（明確に業態がわかる場合のみ）
- 不動産
- 広告 / マーケティング
- ポスティング業

### ❌ LP作成しない
- 業種不明
- 情報不足
- 実態不明

業種不明は必ず「**高（？）**」で管理する。

## 🎨 デザインルール（業種別）

会社ごとに**完全に**変える。

| 業種 | 配色 | 雰囲気 |
|---|---|---|
| リフォーム | 暖色系 / ベージュ / 木目 | 暮らし系・施工事例重視 |
| 工務店 | ネイビー / 木目 | 高級住宅系・職人感 |
| IT企業 | 白 / 青 / 水色 / グレー | SaaS風・コーポレート風 |
| 飲食 | 食欲を刺激する色（赤・黄・茶） | 美味しさが伝わる |
| 美容 | 白 / ピンク / ゴールド | 清潔感・上品 |
| 士業 | 紺 / グレー | 信頼感・堅実 |
| 製造業 | 青 / シルバー | 工業感・先進性 |
| 不動産 | 緑 / ベージュ / 木目 | 安心・信頼 |
| 広告 / マガジン | コーラル / クリーム / グリーン | 編集デザイン |
| ポスティング | オレンジ / 黄 / 紺 | エネルギッシュ・親しみ |

業種ごとに**配色・構成・雰囲気・セクション順**を変更する。

## 📨 DM文章ルール

P列に各社固有の DM 文章を入れる。**必ず**以下を含める：

1. 「勝手ながら貴社専用のホームページ（LP）を作成いたしました。」
2. LP URL
3. 認証情報（ID: sample / PW: sample1234）
4. 公式LINE URL: `https://line.me/R/ti/p/@015vzsdb`

### DM文章テンプレート

```
突然のご連絡失礼いたします。[会社名] ご担当者様
[会社特性に応じた一文]を拝見し、勝手ながら貴社専用のホームページ（LP）を作成いたしました。

【LP URL】[公開URL]
ID：sample
PW：sample1234

[業種別の構成説明1行]
ご感想やご質問など、お気軽に公式LINEまでお寄せください。
公式LINE：https://line.me/R/ti/p/@015vzsdb
```

業種不明・LP未作成社は P列に「業種不明のためDM保留」と記載する。

## 🔄 法人営業リスト作成時のワークフロー

画像のみ渡された場合でも以下を実施：

1. 企業情報調査（社名・法人番号・住所・代表）
2. HP調査（WebSearch + WebFetch）
3. LINE調査
4. スマホ対応調査
5. 営業優先度判定
6. LP作成可否判定
7. LP作成（業種判明の場合）
8. GitHub反映（commit / push）
9. Vercel公開確認（curl）
10. LP URL入力（Q列）
11. **営業優先シート** 用データ生成（A〜Q列フル）
12. **タスク管理シート** 用データ生成（**L列・M列は空白固定**）

## 📊 スプレッドシート2シート運用（必須）

対象：https://docs.google.com/spreadsheets/d/1J4U1infHYW8hxaLSGiRniXU7IVUkVZAQSdUxrT2u_8c/edit

| シート | 入力内容 | L列 | M列 |
|---|---|---|---|
| **営業優先シート** | A〜Q列フル入力 | 営業優先度 | DM送付（=未送付） |
| **タスク管理シート** | 営業優先シートと同じ会社情報 | **空白固定** | **空白固定** |

### タスク管理シート L列・M列 絶対禁止事項
- ❌ 文字を入力しない
- ❌ 数値を入力しない
- ❌ ステータスを入力しない
- ❌ メモを入力しない
- ❌ 空白に見える特殊文字も入れない
- ✅ 完全に空のセルのままにする

→ L列・M列は運用者が手動で管理する領域のため、Claude/AI は一切触れない。

### AI出力フォーマット（2セット必須）

```
【営業優先シート 貼り付け用】
（A〜Q列をTAB区切りで全社分）

【タスク管理シート 貼り付け用】
（営業優先シートと同じ列順／L列・M列のみ空欄で全社分）
```

### Claude の制約（正直に明記）
- Google Spreadsheet を直接編集する権限は持たない
- 「入力」=「貼り付け用データを生成し、運用者が貼り付ける」を意味する
- 2シート分のデータを必ず別ブロックで提示する

## 🌐 URL設計

| URL | 認証 | 内容 |
|---|---|---|
| `/` | なし | STAGE UP公式営業LP |
| `/admin` | admin / stageup-master-2025 | 営業LP一覧 管理画面 |
| `/N-slug` | sample / sample1234 | 各社別 提案 LP |
| `/_proposals/*` | 直接アクセス不可 | LP実体の置き場 |

URL slug 命名規則：`{管理番号}-{英字スラッグ}`（小文字・ハイフン区切り）

## 🔒 トップページの不可侵原則

1. `index.html` は STAGE UP の公式営業LP。**Basic 認証をかけない**
2. 企業別 LP の追加・編集時に `index.html` を改変しない
3. 企業別 LP は必ず `_proposals/` 配下に分離する
4. トップ更新は `index.html` のみを差分編集する
