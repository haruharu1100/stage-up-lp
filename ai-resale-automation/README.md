# AI物販自動化システム(出品候補作成ツール)

仕入れ先の商品データ(CSV)から、**儲かる&売れている&在庫がある商品だけ**を自動で抽出し、
Claude AI が出品タイトル・説明文を作成して、**Googleスプレッドシート/CSVで管理**できるツールです。

> ⚠️ **重要:このツールは「完全自動出品」はしません。**
> 規約違反のリスクを避けるため、**最終的に人間が確認してから出品する**「出品候補リスト作成ツール」です。
> 出品前には必ず各販売サイト(メルカリ・Amazon・ヤフオク等)の最新の規約をご確認ください。

このREADMEは**プログラミング初心者の方がコピペだけで完成できる**ように書いています。
上から順番に進めてください。

## 1. このツールができること

CSVを読み込むと、商品1件ごとに次の処理を自動で行います。

1. **利益計算**:粗利益・利益率を計算
2. **出品判定**:下のルールで6種類に自動分類
   - `出品候補` … 利益率25%以上・販売数1以上・在庫1以上・禁止リスクなし
   - `利益不足` … 利益率が25%未満
   - `需要不足` … 過去1ヶ月の販売数が0
   - `在庫切れ` … 在庫が0
   - `要確認` … 禁止商品リスクあり(人間の確認が必要)
   - `除外` … 赤字(粗利益がマイナス)
3. **AI文章生成**:`出品候補`と`要確認`の商品に、Claude APIで
   - SEO向け商品タイトル
   - 商品説明文
   - 検索キーワード
   - 注意事項
   - 短い商品紹介文
4. **在庫監視**:前回実行時と比べて在庫の増減を表示
5. **出力**:結果をCSVに保存(必要ならGoogleスプレッドシートにも書き込み)

## 3. 必要なもの

| 必要なもの | 説明 | 必須? |
|---|---|---|
| **Python 3.9以上** | プログラムを動かすため | ✅ 必須 |

python --version

## 4. セットアップ手順

### ステップ1:このフォルダに移動する

ターミナル(WindowsはコマンドプロンプトまたはPowerShell)を開き、このフォルダに移動します。

```bash
cd ai-resale-automation
```

### ステップ2:必要なライブラリをインストールする

```bash
pip install -r requirements.txt
```

> うまくいかない場合は `pip` を `pip3` に、`python` を `python3` に変えて試してください。

### ステップ3:設定ファイル(.env)を作る

見本ファイルをコピーして `.env` という名前のファイルを作ります。

- **Windows**:
  ```bash
  copy .env.example .env
  ```
- **Mac / Linux**:
  ```bash
  cp .env.example .env
  ```

### ステップ4:.env を編集する

`.env` をテキストエディタ(メモ帳でもOK)で開き、最低限ここだけ設定します。

```bash
# Claude APIキー(持っている場合)
ANTHROPIC_API_KEY=sk-ant-あなたのキー

# APIキーが無い/まず無料で試したい場合は true にする
USE_AI_OFFLINE_MODE=true
```

> 💡 **APIキーをまだ持っていない場合**は、`USE_AI_OFFLINE_MODE=true` にすれば、
> AIの代わりにテンプレート文章が作られ、**無料で全機能を試せます。**

これでセットアップ完了です!

## 5. 実行手順

### 一番かんたんな実行(サンプルデータで試す)

```bash
python main.py
```

これで `data/sample_input.csv`(10件)を処理し、結果を `data/output/` に出力します。

### よく使うオプション

| コマンド | 何をするか |
|---|---|
| `python main.py` | サンプルCSVを処理する |
| `python main.py --input data/自分のファイル.csv` | 好きなCSVを指定して処理 |
| `python main.py --no-ai` | AI文章生成をスキップ(判定だけ高速確認) |
| `python main.py --only-candidates` | 出力を「出品候補」「要確認」だけに絞る |
| `python main.py --no-sheets` | スプレッドシート書き込みをしない |

### 動作確認テストを実行する(任意)

```bash
python tests/test_basic.py
```

`9/9 件のテストに合格しました。` のように表示されれば正常です。

## 7. 判定ルールの説明

### 利益計算

```
粗利益 = 販売価格 - 仕入れ価格 - 販売手数料 - 送料 - その他費用
利益率 = 粗利益 ÷ 販売価格 × 100
```

### 出品判定(上から順にチェック)

| 優先 | 条件 | 判定 |
|---|---|---|
| 1 | 粗利益がマイナス(赤字) | `除外` |
| 2 | 在庫が1未満 | `在庫切れ` |
| 3 | 過去1ヶ月販売数が1未満 | `需要不足` |
| 4 | 利益率が25%未満 | `利益不足` |
| 5 | 禁止商品リスクあり | `要確認` |
| 6 | 上のどれにも当てはまらない | `出品候補` ✅ |

しきい値(25%・販売数1・在庫1)は `.env` で変更できます。

### 禁止商品リスクの判定

次のどちらかに当てはまると「あり」になります。

- 入力CSVの「禁止商品リスク」列に `あり` と書かれている
- 商品名・型番に禁止キーワード(医薬品・酒・偽ブランド等)が含まれている

禁止キーワードは `config.py` の `PROHIBITED_KEYWORDS` で自由に追加できます。

## 8. Googleスプレッドシート連携(任意)

CSVだけで十分使えますが、スプレッドシートに自動書き込みもできます。

### 手順

1. **Google Cloud Console**(https://console.cloud.google.com/)にアクセス
2. 新しいプロジェクトを作成
3. 「APIとサービス」→「ライブラリ」で **Google Sheets API** と **Google Drive API** を有効化
4. 「APIとサービス」→「認証情報」→「認証情報を作成」→ **サービスアカウント** を作成
5. 作ったサービスアカウントの「キー」→「鍵を追加」→ **JSON** を選んでダウンロード
6. ダウンロードしたJSONを `credentials/service_account.json` に置く
7. 書き込みたいスプレッドシートを用意し、**サービスアカウントのメールアドレス**
   (JSONの中の `client_email`)に**編集者として共有**する
8. スプレッドシートのURL `https://docs.google.com/spreadsheets/d/【ここがID】/edit` のIDをコピー
9. `.env` に次を設定:
   ```bash
   GOOGLE_SERVICE_ACCOUNT_FILE=credentials/service_account.json
   GOOGLE_SPREADSHEET_ID=コピーしたID
   GOOGLE_WORKSHEET_NAME=出品候補
   ```
10. `python main.py` を実行すると、自動でスプレッドシートにも書き込まれます。

> 設定が無い/失敗しても、CSV出力は普通に行われます(止まりません)。

## 9. Google Apps Scriptの使い方

スプレッドシート上で「色分け」「チェックボックス」「出品リスト抽出」「CSV保存」ができます。

### 設定手順

1. 管理用のGoogleスプレッドシートを開く
2. メニューの **拡張機能 → Apps Script** を開く
3. 表示されたエディタの中身を全部消す
4. このプロジェクトの `gas/Code.gs` の中身を**全文コピペ**する
5. 保存(💾ボタン)
6. スプレッドシートに戻って**ページを再読み込み**する
7. メニューに「**物販ツール**」が追加されます

### メニューの使い方

| メニュー | 機能 |
|---|---|
| ① ヘッダーを準備する | A〜Z列の見出しを自動作成 |
| ② 色分け・チェックボックスを設定する | 判定の色分け+X列にチェックボックス |
| ③ 出品OKだけを出品リストへコピー | チェックした行を「出品リスト」シートへ |
| ④ このシートをCSVで保存 | GoogleドライブにCSV保存 |

**使う流れ**:
①でヘッダー作成 → Pythonの出力CSVを貼り付け → ②で色分け →
内容を確認してX列にチェック → ③で出品リスト完成 → 手動で各サイトに出品。

## 10. 入力CSVの作り方

`data/sample_input.csv` をコピーして編集するのが一番かんたんです。

必要な列(**1行目のヘッダー名は変えないでください**):

```
管理番号,商品名,JANコード,型番,仕入れ先,仕入れURL,仕入れ価格,在庫数,販売先,販売価格,販売手数料,送料,その他費用,過去1ヶ月販売数,競合数,禁止商品リスク
```

- 列の**順番は自由**(ヘッダー名で対応づけます)
- 余計な列があっても無視されます
- 数値が空欄なら 0 として扱います
- Excelで作る場合は「**CSV UTF-8(コンマ区切り)**」で保存してください

## 11. エラー時の対処方法

| 症状・エラーメッセージ | 原因 | 対処方法 |
|---|---|---|
| `python: command not found` | Pythonが未インストール | python.orgからインストール。`python3`でも試す |
| `No module named 'anthropic'` 等 | ライブラリ未インストール | `pip install -r requirements.txt` を実行 |
| `CSVファイルが見つかりません` | パスが間違い | ファイル名・場所を確認。`--input`の指定を見直す |
| 文字化けする | 文字コードの問題 | CSVを「UTF-8」で保存。出力は自動でExcel対応(BOM付き) |
| AIが動かず英語のエラー | APIキー誤り/残高不足 | `.env`のキーを確認。困ったら`USE_AI_OFFLINE_MODE=true`に |
| `AI生成に失敗したため…` と出る | API側の一時エラー | 自動でテンプレに切替済み。時間をおいて再実行 |
| スプレッドシートに書けない | 共有/設定不足 | サービスアカウントに編集者で共有したか確認 |
| `PermissionError` | ファイルを開いたまま | 出力先CSVをExcelで開いていたら閉じる |
| 判定が全部「除外」 | 価格データが0 | CSVの販売価格・仕入れ価格が正しいか確認 |

**それでも解決しない場合**:
`python main.py --no-ai --no-sheets` で最小構成にすると、原因の切り分けができます。

## 12. 次に拡張する場合の手順

このツールは「CSVを読んで判定する完成版」です。
将来、次のような自動化を**少しずつ追加**できる設計になっています。

### 拡張A:仕入れ先サイトから自動でデータ取得(スクレイピング/API)

1. `src/` に `supplier_fetcher.py` を新規作成
2. 仕入れ先のAPIまたは(規約で許可された範囲の)スクレイピングで商品データを取得
3. 取得結果を `Product` のリストに変換して返す関数を作る
4. `main.py` の「CSV読み込み」部分を、CSVかAPIかを選べるように分岐させる

> ⚠️ スクレイピングは各サイトの利用規約・robots.txtを必ず守ってください。

### 拡張B:在庫の自動監視(定期実行)

1. すでにある `src/inventory_monitor.py` が在庫差分を記録します
2. これを**定期実行**する(Windowsはタスクスケジューラ、Mac/Linuxはcron)
3. 在庫切れを検知したら、メールやLINE通知を送る処理を追加

### 拡張C:価格監視・自動値付け

1. `src/` に `price_monitor.py` を作成
2. 競合価格を取得し、利益率を保ちつつ最適な販売価格を計算
3. `Product.sell_price` を更新してから判定・出力する

### 拡張D:販売実績データの自動取得

1. 各販売サイトのAPIから「過去1ヶ月販売数」を取得
2. `Product.monthly_sales` に反映してから判定する

### 拡張のコツ

- **部品(`src/`)を1つずつ足す**形にすると壊れにくいです
- 新機能を足したら `tests/test_basic.py` にテストも追加しましょう
- まずは小さく動かして、動いたら次へ。これが完成への近道です
